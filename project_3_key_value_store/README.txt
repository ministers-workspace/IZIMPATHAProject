Thread-Safe In-Memory Key-Value Store
=====================================

Project overview
----------------
This folder contains a complete Java program for a thread-safe in-memory key-value store. It demonstrates:
- Encapsulation with CacheEntry and CacheStore classes
- The this keyword in constructors
- A static global operation counter
- synchronized methods for thread safety
- Multiple concurrent UserThread writers
- A background daemon SnapshotThread
- File output to snapshot.txt
- A Garbage Collection demonstration

How to compile with a full JDK
------------------------------
Open a terminal in the project folder, then run:

    cd src
    javac *.java

How to run after compiling
--------------------------
Still inside the src folder, run:

    java Main YOUR_STUDENT_ID

Example:

    java Main 12345

How C is calculated
-------------------
C is the sum of the digits in the student ID, with a minimum of 5.
For example, student ID 12345 gives C = 1 + 2 + 3 + 4 + 5 = 15, so the program starts 15 concurrent user threads.

If no student ID is provided, the program uses 00000, so C becomes the minimum value of 5.

Alternative run command if javac is unavailable
-----------------------------------------------
Some environments have the Java runtime but not the javac compiler command. In that case, modern Java can run the source file directly from the src folder:

    java Main.java YOUR_STUDENT_ID

Output files
------------
- snapshot.txt is generated by SnapshotThread when the program runs.
- execution_logs.txt contains a sample successful run using student ID 12345.

Important note before final submission
--------------------------------------
If your lecturer requires proof using your own student ID, rerun the program with your real student ID and replace execution_logs.txt and snapshot.txt with the newly generated output.
